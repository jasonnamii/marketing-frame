# Marketing Frame — 마케팅 프레임

> 입력(상황·자료·미션) → 3축 라우팅(시간·고객·실행) → 프레임 3~6개 자동 선택 → 벤토 슬롯 채움 → md+html 듀얼 출력


## Skill Boundaries

- **하는 것** — 마케팅 프레임 라우터 v1.0.
- **안 하는 것** — 카피생성(→copywriting-skill), 피치덱(→bp-guide), 재무모델(→financial-model), UI·랜딩(→ui-designer), 사업전략(→biz-skill), 본질1프레임(→visualize-skill), 단일사례(→benchmark-skill), IMC실행(→brand-campaign), 광고튜닝(→media-buying).

**별칭 흡수**: 마케팅프레임 = 마케팅엔진 = 프레임라우터 = 벤토마케팅 = 한장마케팅. 마케팅진단·마케팅기획·캠페인진단·캠페인기획 모두 본 스킬로 라우팅. 클래식 4P·STP·SWOT 부터 디지털 AARRR·JTBD·CPF, 메타 Cynefin·OODA·Wardley·Strategy Diamond 까지 140+ 프레임 DB 보유.

기존 스킬과 경계: **프레임 라우터 전용**. 프레임 도출까지만 담당, 카피·BP·재무·UI 등 실행은 위임.

## When to Use

- 사용자가 "진단해줘", "기획해줘", "라우팅해줘", "벤토로 만들어줘", "diagnose" 같은 표현으로 발동
- 캠페인 점검 시, 신제품 출시 전, 성장 정체 진단 시.
- **안 쓸 때** — 카피생성(→copywriting-skill), 피치덱(→bp-guide), 재무모델(→financial-model), UI·랜딩(→ui-designer), 사업전략(→biz-skill), 본질1프레임(→visualize-skill), 단일사례(→benchmark-skill), IMC실행(→brand-campaign), 광고튜닝(→media-buying).

## 절대 규칙

```
[NO_WORK_LABEL] §STYLE [CONTENT]
RULE      : 산출물·대화 = 인간 언어. 작업 라벨 ZERO.
판정      : "이 대화 밖 사람이 사전 없이 읽나?" NO → 금지
ALLOW     : 업계용어(BEP·KPI·MECE·MVP) · 고유명사 · 법조문
CONVERT   : Y2→"2년차" / 4축→실제 4개 이름 / C:E:W:→평문
SELF_CHECK: 산출 직전 스캔. 1개 발견 = 차단·재작성
```

## 발동

P1·P2 단어 hit OR 마케팅 기획·진단 맥락 + 다음 중 1개:
- 미션·상황·자료 제시
- "어떤 프레임으로 봐야 할지" 질문
- AS-IS / TO-BE / Pain Points / Strategic Questions / FAQ Gap 단어 등장
- 벤토 형식 마케팅 분석 요청

## Phase 0: 미션 패킷 (핑퐁 1회)

5요소 미수집시 1회만 핑퐁. 부족시 가설로 채우고 Confidence ≤50.

| 요소 | 질문 |
|---|---|
| MODE | 기획? 진단? |
| DOMAIN | SaaS·D2C·콘텐츠·금융·리테일·플랫폼·스타트업·브랜드 |
| TARGET | 누구를 위한 마케팅? |
| OBJECTIVE | 무엇을 달성/해결? |
| CONSTRAINT | 시간·예산·조직 제약 |

## Phase 1: 모드 자동감지

- "기획·설계·만들어·전략·런칭·캠페인" → **PLAN**
- "진단·분석·왜·복기·점검·이슈" → **DIAGNOSE**
- 둘 다 → 사용자 1회 확인

| 항목 | PLAN | DIAGNOSE |
|---|---|---|
| 입력 | 미션·가설·타겟 | 현황·자료·이슈 |
| 슬롯 톤 | 가설형·제안형 | 현상기술·근거인용 |
| Pain | 잠재 리스크 | 관찰된 문제 |
| AS-IS / TO-BE | 향후 vs 목표 | 현재 vs 이상 |
| One Promise | forward 약속 | backward 본질 |
| Confidence | 50~70 | 70~90 |

### MECE 검사 (필수)
- 시간축 2 이상 분포 / 고객축 3 이상 단계 / 실행축 2 이상 페이즈
- 동일 카테고리 중복 ✗ (AARRR vs RARRA, ICE vs RICE 중 1택)

## Phase 4: 벤토 슬롯 (10개)

```
[1] Strategic Questions (3~5)         ← McKinsey Issue Tree
[2] 3축 라우팅 결과                    ← 시간·고객·실행 매핑 표
[3] 선택 프레임 카드 (3~6개)            ← name + 1줄정의 + WHY
[4] Pain Points (6개)                  ← PAS / CRITICAL·HIGH·MEDIUM
[5] AS-IS / TO-BE 표                   ← 현재 vs 목표
[6] Additional Issues                  ← 경쟁·타깃·SEO·CX 보완
[7] FAQ Gap Analysis                   ← 사용자 질문 역설계
[8] Counter-Metric 가드                ← NSM 과적화 방지
[9] One Promise                        ← 단일 약속/본질
[10] Confidence + Source               ← UP §CONFIDENCE
```

각 슬롯 5~12줄. 위계: `→ references/bento-template.md`

## Phase 4.5: PRE_WRITE — 슬롯 채우기 직전 룰

**목적:** 벤토 10슬롯을 *작성 시점*에 단문·수치+출처·단언형 강제. 한장 마케팅 = AI식 흔적 1줄도 즉시 사망.

### 5룰 (슬롯 채우기 직전 강제)

| # | 룰 | 벤토 결로 변환 | FAIL 신호 |
|---|---|---|---|
| 1 | **슬롯 헤드 단문** | 슬롯 제목 ≤15자. 본문 1행 ≤30자 | 라벨식·30자+ |
| 2 | **One Promise 단일** | [9] One Promise = 1줄 ≤25자. 균형요약 ✗ | "A하면서 B도" |
| 3 | **수치+출처 강제** | Pain·AS-IS·TO-BE·Counter-Metric 모두 수치 + 출처 | "큰 문제" 추상 |
| 4 | **문장당 1명제** | 1행 = 1주장 | AND·OR 묶기 |
| 5 | **AI식 사전회피** | "고객 중심의", "혁신적인 접근", "전략적 통찰" — 작성 단계 차단 | 컨설팅 관용어 |

### 슬롯별 PRE_WRITE

| 슬롯 | 길이 | 형식 |
|---|---|---|
| Strategic Questions | 1Q ≤30자 | 의문문 |
| Pain Points | 1P ≤25자 | 단언형·우선순위 |
| AS-IS / TO-BE | 1행 ≤25자 | 수치 대비 |
| FAQ Gap | 1Q ≤30자 | 의문문 |
| Counter-Metric | 1지표 ≤20자 | 수치 + 임계점 |
| One Promise | ≤25자 | 단일 약속 |

### 자체검증 (Phase 5 출력 직전)

| # | 체크 | 위반 |
|---|------|------|
| 1 | 슬롯 헤드 ≤15자, 단언형? | 초과·라벨식 = 재작성 |
| 2 | One Promise 단일·≤25자? | 다중 = 단일 재수렴 |
| 3 | 수치+출처? | 누락 = 자료 보완 |
| 4 | AI식 어휘 hit? | hit ≥ 1 = 평문 변환 |

## Phase 4.7: §CONFIRM_GATE — 송출 직전 형 컨펌 (3단계 가드)

**목적:** PRE_WRITE 자가신고 우회 차단. 자체검증 통과 = 송출 ✗ → 형 컨펌 후 송출.

**발동:** 산출물 송출 *직전* 1회.

**형식 (verbatim):**
```
🔍 송출 전 검토 부탁드려요. AI 티·번역투·장황 있나요?
[OK / 수정 / 재작성]
```

**규칙:**
- 형 OK → 최종 송출
- 형 수정 → 형 지적 부분만 PRE_WRITE 재진입 → 재출력
- 형 재작성 → 모드 처음부터 재실행

**SCOPE_OUT (컨펌 면제):**
- ≤5줄 답변·단답·확인성·실행보고
- 진단 본문 (대화 내 분석, 산출물 ✗)
- 일반대화·핑퐁·컨펌게이트 자체

**CHECK:** 산출물 송출인데 컨펌게이트 미발동 = FAIL → 송출 폐기·재출력.

## Examples

### 예시 1 (PLAN, SaaS B2B)
입력: "시리즈 A 직전, 신규 가입↑·28일 유지율 18%로 낮음. 다음 분기 성장 전략."
- 모드: PLAN
- 도메인: SaaS B2B
- 라우팅: 시간 Complex(중기) · 고객 Activation→Retention · 실행 최적화+스케일
- 선택 프레임: Growth Loops · AARRR · NSM+Counter · Reverse Trial · OODA
- One Promise: "유지율 18% → 35%로 끌어올리는 활성화 루프"

### 예시 2 (DIAGNOSE, D2C)
입력: "뷰티 D2C 매출 정체. 광고비↑·CAC↑. 인플루언서·UGC도 효과 없음."
- 모드: DIAGNOSE
- 도메인: 이커머스 D2C
- 라우팅: 시간 Complicated(중기) · 고객 인지 포화→검토 무력 · 실행 최적화+재진단
- 선택 프레임: HEAVY-LIGHT · Four Realms · Cohort · RFM
- One Promise: "Heavy 충성층 함정에서 Light Buyer 확장으로 성장축 전환"
